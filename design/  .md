# Hygiea — Design Brief for Enhanced Desktop Layout

## What is Hygiea?

A spiritual-hygiene and astrology web app rooted in Rudolf Steiner's Anthroposophy. It gives the user a living instrument to understand themselves through the natal chart, the seven planetary spheres, daily transits, and the cosmological biography arc. The tone is contemplative, precise, and grounded — not mystical kitsch.

**Live URL:** sukoon.arabic-astro.com (Sukoon is the Arabic sister app; Hygiea is the English version)
**Stack:** Next.js 15 App Router, Tailwind CSS v4, Framer Motion, DM Sans font

## Current Layout (The Problem)

The app was built mobile-first (max-width 430px). On desktop, the content just stretches full-width with no structure. The left sidebar is 220–272px. The rest is a single content column — lots of empty white space on the right.

**Current AppShell main:**
```
flex-1 w-full max-w-[430px] mx-auto → md:max-w-full md:mx-0
```

---

## Desired Desktop Layout: 4-Column Grid

The enhanced desktop layout should use a **4-column CSS grid** spanning the full content area (right of sidebar).

### Column Roles
| Col | Width | Contents |
|---|---|---|
| 1 | ~260px | **Context panel** — chart summary, active transits, quick stats about the current section |
| 2–3 | flex (main content) | **Primary content** — the page's main feature (chart wheel, day cards, sphere list, etc.) |
| 4 | ~240px | **Detail / helper panel** — tapped item detail, notes, the "?" helper panel |

On medium screens (md, 768–1199px): collapse to 2 columns (context panel hidden, detail panel hidden).
On large screens (lg, 1200px+): show all 4 columns.
On mobile: single column as today.

### Visual Rules
- Column gaps: 1px `rule-soft` dividers, no heavy borders
- Column backgrounds: all `cream`, context panel can be `cream-soft`
- Sticky column headers at top of each column (like a spreadsheet header row)
- No cards floating in empty space — content should feel dense and purposeful

## Pages to Redesign

### 1. Self (`/self`) — HIGHEST PRIORITY
**Current:** Single form asking for birth data, then chart wheel below  
**Desired desktop:**
- Col 1: Birth data summary + planet list (Sun ♌ Leo, Moon ♓ Pisces, etc.)
- Col 2–3: Natal chart wheel (SVG, large, centered)
- Col 4: Tapped planet detail — sign, house, meaning, aspects

### 2. Day (`/today`) — HIGH PRIORITY
**Current:** Single-column card stack  
**Desired desktop:**
- Col 1: Date, Moon phase, today's planetary ruler, active transits count
- Col 2–3: Day cards stacked vertically (Transit · Body · Two Winds · Question · Learning)
- Col 4: Expanded card detail when one is tapped

### 3. Spheres (`/spheres`) — HIGH PRIORITY
**Current:** List of 7 spheres  
**Desired desktop:**
- Col 1: Sphere list (Moon, Mercury, Venus, Sun, Mars, Jupiter, Saturn)
- Col 2–3: Selected sphere deep-dive (description, exercises, meditation)
- Col 4: Today's sphere highlight, planetary hour

### 4. Orrery (`/orrery`) — MEDIUM PRIORITY
**Current:** Full-screen 3D R3F canvas  
**Desired desktop:** Keep the 3D canvas in col 2–3, use col 1 for planet list + controls, col 4 for selected planet info panel

## Inspiration / Aesthetic Direction

- **myBodyGraph.com** — clean 2-column with form + chart side by side, very structured
- **Notion-style panels** — quiet borders, no drop shadows, information density without clutter
- **Scientific instrument** — the UI should feel like a precision tool, not a consumer app. Calm, spacious but intentional.

The brand phrase: *"A living scientific-spiritual instrument."*

---

## What NOT to Do

- No drop shadows on cards (use `ring-1 ring-rule-soft` instead)
- No gradients on backgrounds (gradients only on cosmic/3D elements)
- No emojis in UI
- No dark mode
- No rounded corners larger than 22px on cards
- Don't hide content behind tabs on desktop if it can be shown side-by-side
